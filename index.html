Documentação para desenvolvimento de temas da plataforma de e-commerce Tray

Tray OpenCode (Beta)

Criar novos layouts para sua loja agora é um processo mais simples e rápido. Utilizando o Tray OpenCode você pode fazer vários testes de layout.

Com o Tray OpenCode você tem uma ótima oportunidade de deixar sua loja virtual sempre bonita e atualizada para várias ocasiões que envolvem datas comemorativas como Natal, Dia das Mães e Dia dos Namorados, entre outras.

Avisos

Importante: Não nos responsabilizamos pelo uso de variáveis, helpers e funções que não estão documentadas aqui. Elas podem passar por ajustes e mudanças sem aviso prévio.

Seja cuidadoso ao utilizar variáveis ou parâmetros que estão com a nomenclatura em português, provavelmente será alterado para o inglês assim que possível. Avisaremos com antecedência sobre essas mudanças.

Iniciando meu primeiro tema

Existem algumas maneiras disponíveis para iniciar um tema, veja algumas delas.

Opção 1: OpenCode Theme GEM (Recomendado)

A GEM permitirá que você tenha um ambiente de alta produtividade, por isso é a nossa recomendação.

Pré-requisito: Ter o Ruby instalado.

Para que as gems funcionem corretamente no MAC, é necessário instalar o XCode: Instalar XCode via AppStore

Mais informações poderão ser encontradas nesse link: Setting up a Ruby development environment on Mac OS

Download: Tray OpenCode GEM

Instalação:

$ gem install opencode_theme-0.0.1b.gem
$ opencode -h

Aqui são exibidas todas as opções da GEM:

Para criar seu primeiro tema, execute os seguintes comandos:

$ mkdir primeiro-tema
$ cd primeiro-tema
$ opencode bootstrap API_KEY PASSWORD 'PrimeiroTema'

Após realizar o bootstrap, ele baixará todos os arquivos pra você começar.

Após isso, execute: $ opencode watch

Sempre que salvar um arquivo, ele realizará o upload automaticamente:

Opção 2: Git

Você poderá clonar o nosso repositório, executando o seguinte comando:

git clone git@github.com:tray-tecnologia/theme-template.git nome-tema

O upload deverá ser feito via interface, veja na opção 3.

Opção 3: Download/upload via interface

Você consegue fazer várias operações (inclusão, upload, download, publicação) pela nossa interface. A imagem a seguir é a tela principal do Tray OpenCode.

O primeiro passo é "Incluir um novo layout":

Após isso, poderá baixar os arquivos, alterá-los e realizar o upload dos mesmos:

Então, poderá clicar em "Visualizar layout" para conferir como seu tema está ficando.

Estrutura do tema

Para manter o desenvolvimento dos temas bem organizado, disponibilizamos uma estrutura enxuta e simples para seus arquivos. Veja a estrutura inicial:

elements/ -> trechos de código e componentes reutilizáveis
layouts/  -> estrutura do seu tema, código que será carregado em todas as páginas
pages/    -> conteúdo de cada página da sua loja
css/      -> você pode inserir suas folhas de estilo e webfonts
js/       -> você pode inserir seus plugins js
img/      -> pasta de imagens
configs/  -> opções da sua loja virtual

Nas pastas "elements", "css", "js" e "img" você poderá criar os arquivos da maneira que achar melhor.

Porém, nas pastas "layouts" e "pages" você precisará criar o arquivo com o nome correto. Veja os arquivos que você pode criar:

layouts/
    ajax.html (páginas que são carregadas via ajax)
    default.html (todas as páginas, exceto as carregadas via ajax)
    error.html (crie sua página de erro personalizada)
pages/
    home.html (página inicial)
    map.html (mapa do site)

Obs: Essa lista de arquivos disponíveis será atualizada constantemente.

Codificação de arquivos

Utilize a codificação dos seus arquivos sempre ISO-8859-1.

Se estiver utilizando o Sublime Text, adicione as seguintes opções no Settings - User:

"fallback_encoding": "Western (ISO 8859-1)",
"default_encoding": "Western (ISO 8859-1)",

Variáveis e Helpers

Todas as variáveis utilizam o seguinte padrão: myVariable.variable_key. Para exibir o nome do produto, você simplesmente fará a seguinte chamada: {{ product.name }}. Mas, o que são esses caracteres {{ }}?

Nós utilizamos um Template Engine para renderização das views. Você precisará lembrar apenas de dois tipos de chamadas: {{ }} e {% %}. Veja os exemplos de utilização abaixo:

Exemplos

Imprimindo variáveis:

{{ product.name }}

if:

{% if product.name %}
    {{ product.name }}
{% endif %}

for:

{% for image in product.images %}
    <img src="{{ image.src }}" alt="{{ product.name }}">
{% endfor %}

Declarando variáveis:

{% set company = 'Tray' %}

Se quiser mais informações sobre a utilização do Template Engine, visite a seguinte página: Documentação - Twig

Banners

Para imprimir um banner, você poderá utilizar o helper de banner, utilizando a seguinte chamada:

{{ banner.home }}
{{ banner.footer }}

home e footer são as posições em que o banner foi cadastrado. Veja a lista de posições disponíveis: home, footer, bottom, side, title, floating, popup, extra1, extra2, extra3, extra4

Se você deseja criar um banner da sua maneira, poderá trabalhar com as seguintes informações:

Chaves Descrição
id Identificador único do banner
type Tipo do banner, valores possíveis: "javascript", "image", "flash", "gallery"
description Descrição do banner
src URL do banner
link Link do banner (apenas nos tipos "javascript" e "image")
width Largura da imagem ou do objeto flash
height Altura da imagem
alt_text Texto alternativo para a imagem do banner
target Se o link vai abrir em uma nova janela ou na mesma
banner_alt Banner alternativo caso não tenha suporte ao flash (apenas quando o tipo for "flash"), possui as chaves "src", "link", "target"
margin_top Especifica a posição do banner referente ao topo da página (apenas banner "floating")
margin_left Especifica a posição do banner referente ao lado esquerdo da página
Exemplos

Lembrando que o padrão de chamada é sempre: banners.{position}.{key}. Veja exemplos utilizando a posição home:

{{ banners.home.id }}
{{ banners.home.type }}
{{ banners.home.description }}
{{ banners.home.src }}
{{ banners.home.link }}
{{ banners.home.width }}
{{ banners.home.height }}
{{ banners.home.alt_text }}
{{ banners.home.target }}
{{ banners.home.banner_alt.src }}
{{ banners.home.banner_alt.link }}
{{ banners.home.banner_alt.target }}
Banner JS

O banner JavaScript possui algumas informações que os outros banners não tem, segue a lista:

Chaves Descrição Valores
interval Intervalo da transição entre cada slide 3 a 10 (segundos)
velocity Velocidade da animação 0.5 | 1.0 | 1.5 | 2.0
animation Tipo da animação blind | block | cube | cubeSpread | fade | fadeFour | glassCube | horizontal | tube
stop_over Pausar a troca de slides quando o mouse estiver sobre o banner 0 | 1
navigation Tipos de navegação 0 | 1 | 2
progressbar Exibir barra de progresso 0 | 1
slides Dados das imagens cadastradas image | link | label | target | width | height
Exemplos Banner JS 
{{ banners.extra1.interval }}
{{ banners.extra1.velocity }}
{{ banners.extra1.animation }}
{{ banners.extra1.stop_over }}
{{ banners.extra1.navigation }}
{{ banners.extra1.progressbar }}
{% for slide in banners.extra1.slides %}
    <a href="{{ slide.link }}" target="{{ slide.target }}">
        <img src="{{ slide.image }}" alt="{{ slide.label }}" />
    </a>
{% endfor %}

Products

Informações sobre os produtos cadastrados no Painel Administrativo.

Buscando produtos

Para buscar produtos de acordo com as suas necessidades, utilize a seguinte chamada:

{% set products = Products({
    'filter': 'featured',
    'limit': 8,
    'order': {'quantity_sold': 'desc'},
    'categories': 1,
    'brands': 'Tray'
}) %}

A chamada acima retornará até 8 produtos em destaque ordenados pelos mais vendidos que estejam cadastrados na categoria 1 e marca Tray.

{% set products = Products({
    'filter': 'new'
}) %}

A chamada acima retornará os produtos marcados como lançamento. Veja mais parâmetros disponíveis abaixo:

Parâmetros disponíveis (todos os parâmetros são opcionais)
Parâmetro Descrição e opções
filter featured, new, free_shipping, promotion, available, rand
limit Quantidade máxima de produtos que será retornada
order id, name, price, created
categories ID da categoria
brands String da marca

Obs: É possível realizar múltiplas ordenações, conforme exemplo:

{% set products = Products({
    'order': {
        'quantity_sold': 'desc',
        'name': 'asc',
        'id': 'desc'
    }
}) %}

As requisições de produtos sempre retornarão um array de dados onde cada chave está descrita abaixo.

Chaves Descrição
id Identificador único do produto
name Nome do produto
category_id Identificador único da categoria principal do produto
price Valor do produto
price_offer Valor do produto em promoção
payment Retorna o parcelamento do produto
link Link para a página do produto
available Disponibilidade do produto
stock Quantidade de produtos em estoque
featured Retorna verdadeiro se for destaque
new Retorna verdadeiro se for lançamento
free_shipping Retorna verdadeiro se possuir frete grátis
additional_button Retorna verdadeiro se possuir botão adicional cadastrado

Filter

Nas páginas de catálogo e busca, você poderá filtrar os produtos exibidos e ordená-los. Quando adicionar o atributo data-form="filter" em um formulário, ele iniciará o módulo filter. Esse módulo fará com que sempre que um <select> for alterado (onchange), o formulário seja submetido e recarregue os produtos.

Exemplos
<form data-form="filter">
    <select></select>
</form>
